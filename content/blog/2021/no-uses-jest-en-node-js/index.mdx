Llevo tiempo leyendo quejas desde la comunidad y miembros del core de Node.js sobre Jest y que recomiendan no usarlo para aplicaciones en Node.js. Hasta ahora siempre he usado Jest porque me parece una herramienta muy completa y cómoda. Además, como cuando hago frontend uso React también uso Jest, ya que es el test runner que viene con Create React App.

Hace un par de meses probé tap, que es lo que recomiendan como alternativa a Jest para hacer test en Node.js. Sin embargo, aunque me pareció un gran herramienta, no me terminó de convencer la forma de organizar los test y como hasta ahora no me he encontrado ningún problema con Jest, pues no le di mayor importancia.

Este [issue en Github de serverless](https://github.com/serverless/serverless/pull/6517) comenta las razones por las que acabaron pasando de Jest a Mocha. Es cierto que es un issue de 2019, pero te da una idea de el por qué. Además, he visto a Matteo Collina comentar una y mil veces que el problema que tiene con Jest es que no estás testeando tu aplicación corriendo en Node.js, sino corriendo en Jest.

HABLAR DE LA EXPERIENCIA PROPIA

PROPONER ALTERNATIVAS CON TAP Y MOCHA
